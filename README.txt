---------
概要 ver 20211001
---------
本ソフトウェアはNCOMの提供する050plusのSIPプロトコル
を確認するための試験ツールです．通話はできません．
050plusのSIPはTLS上で動作するため，ネットワークモニタ
などでは，実際にどのようなパケットがやり取りされているのか
確認できません．本ツールは送受信SIPパケットを標準出力に
表示します．試験ツールや製品開発，教育用に使用してください．
本ツールはREGISTERおよびINVITEメッセージのみ確認できます．
呼を成立させませんので課金されないかわりに，BYEなどのメッセージ
内容は確認できません．
----------
動作
----------
050plusのSIPクライアントとして
REGISTERおよび着信のためのINVITE受信試験を
行います．
050plus正規のアカウントを取得して，confファイルを
設定してください．(sip.conf.defをsip.confに改名して
ファイル中の*****部分を適切な文字列で置き換えてください)
fmc番号やsipdID,プロキシの取得方法はインターネット上の
手法で取得してください．
そして本コマンドを起動してSIPプロキシーサーバ間で
TLSセッションが確立するとREGISTER
を発信します．認証が成功するとREGISTER状態になります．
その状態で，他の一般電話からこのアカウントの電話番号に
発呼するとINVITEが受信できます．
本試験ツールは INVITEにはエラーを返送しますので，発呼は失敗して通話
できません．課金もされません．
コントロールＣで終了してください．
----------
ご注意1
----------
050plusのSIPサーバの中間証明書を含むサーバ証明書の検証のために
CN=Baltimore CyberTrust Rootのルート証明書を同梱しています．
Baltimore.cer
CONFファイルにこのファイルが設定されているとSIPサーバ証明書の
検証を実施します．CONFファイルからCACERTで始まる行を削除すると
本ファイルは読み込まれません．ルート証明書を読み込めない設定
がなされていると，本ツールはサーバ証明の検証を行いません．
どのような証明書が与えれようとも，その公開鍵を信じてTLSセッションを
開始します．
----------
ご注意1
----------
本ソフトウェアはNTTコム非公認です．用途として
050plusのSIPサーバとの確認試験に使えますが，その用途専用では
ありません．その他のSIPサーバとの接続確認試験にも使えます．
----------
Make & Run
----------
Makeにはopensslの開発者用ライブラリが必要です．
事前にインストールをしておいてください．
本プログラムは Ubuntu上で動作を確認しています。
ファイルを解凍したら、
makeコマンドでプログラムをコンパイルしてください。
confファイルを設定後，sipcで起動します．
